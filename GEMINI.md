# PDF 簽名系統開發規格書

##  系統概述

**目標**：建立一個可視化的PDF簽名框編輯器，讓使用者可以在PDF文件上拖拽放置簽名區域，然後進行數位簽名。

**核心理念**：避免每次都要重寫HTML，而是提供一個通用的簽名框配置工具。

---

##  功能架構

### 1. 主要介面區域

#### 1.1 頂部工具列
- **編輯模式按鈕**：切換到編輯模式，可以拖拽和配置簽名框
- **預覽模式按鈕**：切換到預覽模式，可以實際進行簽名操作
- **載入PDF按鈕**：上傳並載入PDF檔案
- **匯出設定按鈕**：將簽名框配置匯出為JSON檔案
- **清除全部按鈕**：清除所有已放置的簽名框

#### 1.2 左側元件面板
- **簽名元件庫**：包含可拖拽的元件
  - 簽名框：用於手寫簽名
  - 日期框：自動填入日期
  - 文字框：可輸入自訂文字
- **已放置欄位列表**：顯示目前文件上的所有簽名框
  - 每個欄位顯示類型和標題
  - 提供編輯和刪除按鈕

#### 1.3 右側文件檢視區
- **PDF檢視區域**：顯示載入的PDF或範例文件
- **拖放區域**：接受從左側拖拽過來的元件
- **簽名框顯示**：顯示已放置的簽名框，在編輯模式下可以移動和調整大小

##  核心流程

### 2. 編輯模式流程

#### 2.1 載入PDF文件
1. 使用者點擊「載入PDF」按鈕
2. 開啟檔案選擇對話框，限制只能選擇PDF檔案
3. 讀取PDF檔案內容（使用FileReader API）
4. 載入PDF.js函式庫（如果尚未載入）
5. 使用PDF.js解析並渲染PDF第一頁
6. 調整檢視區域尺寸以配合PDF頁面
7. 在PDF背景上覆蓋透明的拖放區域

#### 2.2 拖拽放置簽名框
1. 使用者從左側元件面板選擇要放置的元件（簽名框/日期框/文字框）
2. 按住滑鼠左鍵開始拖拽
3. 拖拽過程中元件變為半透明，顯示拖拽狀態
4. 滑鼠移到右側PDF區域時，拖放區域高亮顯示
5. 放開滑鼠時，在放置位置創建對應的簽名框
6. 新建的簽名框具有以下特性：
   - 藍色虛線邊框，表示可編輯狀態
   - 預設尺寸（簽名框：180x60px，日期框：200x30px，文字框：150x30px）
   - 右上角顯示編輯和刪除按鈕
   - 右下角顯示調整尺寸控制點

#### 2.3 簽名框操作
1. **移動簽名框**：
   - 點擊簽名框任意位置（除了控制按鈕）
   - 拖拽到新位置
   - 自動限制在PDF頁面邊界內

2. **調整簽名框大小**：
   - 點擊右下角的調整尺寸控制點
   - 拖拽改變寬度和高度
   - 設定最小尺寸限制

3. **編輯簽名框屬性**：
   - 點擊編輯按鈕（鉛筆圖示）
   - 彈出對話框修改顯示文字
   - 更新簽名框標題

4. **刪除簽名框**：
   - 點擊刪除按鈕（垃圾桶圖示）
   - 確認刪除對話框
   - 從文件和列表中移除

#### 2.4 簽名框資料管理
1. 每個簽名框分配唯一識別碼（field_時間戳）
2. 記錄簽名框屬性：
   - ID、類型、位置座標、尺寸
   - 顯示文字、簽名狀態、簽名資料
3. 即時更新左側欄位列表
4. 所有變更即時儲存到記憶體陣列

---

### 3. 預覽模式流程

#### 3.1 模式切換
1. 使用者點擊「預覽模式」按鈕
2. 隱藏所有編輯控制項（編輯按鈕、調整大小控制點）
3. 簽名框邊框改為實線，表示可操作狀態
4. 拖放區域隱藏
5. 狀態指示器顯示「預覽模式」

#### 3.2 簽名操作流程
1. **觸發簽名**：
   - 使用者雙擊簽名框
   - 開啟全屏簽名彈窗

2. **簽名彈窗介面**：
   - 標題列：顯示「數位簽名」和關閉按鈕
   - 提示資訊：說明簽名操作方式
   - 簽名畫布：740x250px白色背景區域
   - 控制面板：筆觸設定和操作按鈕

3. **簽名繪製**：
   - 支援滑鼠繪製
   - 支援觸控筆/手指觸控
   - 即時顯示筆跡
   - 筆觸設定：
     - 顏色選擇：黑色、藍色、紅色
     - 粗細調整：1-8px可調

4. **簽名編輯功能**：
   - 清除按鈕：清空整個畫布
   - 復原按鈕：撤銷最後一筆
   - 確認按鈕：保存簽名並關閉彈窗

5. **簽名保存流程**：
   - 將Canvas內容轉換為Base64圖片格式
   - 更新對應簽名框的資料記錄
   - 在文件上顯示簽名圖片
   - 簽名框狀態改為「已簽名」（綠色邊框）

##  資料結構設計

### 4. 簽名框資料結構
```
signatureField = {
    id: "field_1627123456789",           // 唯一識別碼
    type: "signature",                   // 類型：signature/date/text
    x: 100,                             // X座標（相對於PDF頁面）
    y: 200,                             // Y座標
    width: 180,                         // 寬度
    height: 60,                         // 高度
    text: "簽名區域",                    // 顯示文字
    signed: false,                      // 是否已簽名
    signatureData: null                 // Base64簽名圖片資料
}
```

### 5. 匯出設定格式
```
exportConfig = {
    version: "1.0",                     // 設定版本
    timestamp: "2025-07-29T10:30:00Z",  // 建立時間
    pdfInfo: {                          // PDF資訊
        filename: "contract.pdf",
        pageCount: 1,
        pageSize: {width: 794, height: 1123}
    },
    fields: [                           // 簽名框陣列
        // 包含所有簽名框資料
    ]
}
```

##  技術實作要點

### 6. 拖拽功能實作
1. **HTML5 Drag & Drop API**：
   - 元件設定 `draggable="true"`
   - 監聽 `dragstart`, `dragover`, `drop` 事件
   - 使用 `dataTransfer` 傳遞拖拽資料

2. **事件處理順序**：
   - dragstart → 記錄拖拽元件資訊
   - dragenter → 高亮拖放區域
   - dragover → 允許放置（preventDefault）
   - drop → 建立簽名框
   - dragend → 重置元件外觀

### 7. PDF處理實作
1. **PDF.js整合**：
   - 從CDN載入PDF.js函式庫
   - 設定Worker路徑
   - 使用Promise處理非同步載入

2. **PDF渲染流程**：
   - FileReader讀取PDF檔案
   - pdfjsLib.getDocument解析PDF
   - getPage獲取頁面物件
   - 在Canvas上渲染頁面內容

### 8. 畫布簽名實作
1. **Canvas繪圖**：
   - 監聽mouse和touch事件
   - 記錄筆跡路徑點陣列
   - 支援復原功能（儲存筆劃歷史）

2. **簽名資料處理**：
   - Canvas.toDataURL()轉換為Base64
   - 壓縮和最佳化圖片大小
   - 儲存到對應欄位資料

##  實際應用流程

### 9. 完整使用流程

#### 9.1 管理員設定流程
1. 開啟PDF簽名編輯器
2. 點擊「載入PDF」上傳合約範本
3. 切換到「編輯模式」
4. 從左側拖拽「簽名框」到合約中需要簽名的位置
   - 甲方簽名位置
   - 乙方簽名位置
   - 見證人簽名位置
5. 拖拽「日期框」到日期位置
6. 拖拽「文字框」到需要填寫的位置
7. 調整各簽名框的大小和位置
8. 點擊「匯出設定」保存配置檔案

#### 9.2 使用者簽名流程
1. 系統載入PDF檔案和簽名框配置
2. 切換到「預覽模式」
3. 雙擊需要簽名的簽名框
4. 在彈出的簽名面板中進行手寫簽名
5. 確認簽名，系統將簽名顯示在文件上
6. 重複步驟3-5完成所有必要簽名
7. 系統生成包含所有簽名的完整文件

#### 9.3 後端整合流程
1. 前端將簽名資料（Base64）發送到後端
2. 後端將簽名資料儲存到資料庫
3. 需要時，後端讀取PDF範本和簽名資料
4. 將簽名合成到PDF上生成最終文件
5. 提供文件下載或歸檔功能

##  使用者體驗設計

### 10. 視覺回饋
1. **拖拽狀態**：元件拖拽時半透明顯示
2. **拖放區域**：滑鼠進入時高亮藍色邊框
3. **簽名框狀態**：
   - 編輯中：藍色虛線邊框
   - 已簽名：綠色實線邊框
   - 選中狀態：陰影和控制按鈕顯示

### 11. 操作便利性
1. **鍵盤快捷鍵**：
   - ESC：關閉簽名彈窗
   - Ctrl+Z：復原簽名筆劃
   - Ctrl+S：保存簽名
2. **滑鼠操作**：
   - 單擊：選擇簽名框
   - 雙擊：開啟簽名功能
   - 拖拽：移動簽名框
3. **觸控支援**：支援平板和觸控螢幕操作

##  效益分析

### 12. 解決的問題
1. **重複開發**：避免每個PDF都要重寫HTML簽名頁面
2. **維護困難**：統一的配置格式便於維護
3. **使用者體驗**：視覺化拖拽比程式碼配置更直觀
4. **靈活性**：可以快速調整簽名框位置和數量

### 13. 技術優勢
1. **前端實現**：不依賴後端，減少伺服器負載
2. **標準化**：基於HTML5標準API，相容性好
3. **模組化**：簽名功能可以獨立使用
4. **擴展性**：容易添加新的欄位類型

### 14. 應用場景
1. **合約簽署**：各種商業合約的數位簽名
2. **表單填寫**：政府表單、申請書等
3. **文件審批**：內部文件的審批流程
4. **教育應用**：作業批改、成績確認等

---

這個系統的核心價值是提供一個**通用的簽名框配置工具**，讓你不需要為每個不同的PDF都重寫程式碼，而是透過視覺化的方式快速配置簽名位置，大大提高開發效率！
